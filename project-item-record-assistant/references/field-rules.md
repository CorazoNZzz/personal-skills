# 字段与业务规则

## 目标字段顺序
写入金山文档时，只能按以下 15 个字段追加一行：

编号、日期、项目名称、事项类型、事项描述、当前状态、优先级、责任方、协同方、下一步动作、截止时间、留痕方式、留痕位置、是否写入周报、备注

## 两步式流程
第一步只生成草稿，编号固定为 `待生成`，不得写入。第二步必须在用户明确确认后才写入。

确认指令包括：确认、确认写入、可以写入、没问题写进去、记录进去、加进去、写入、保存到台账。

取消指令包括：取消、不要写了、先不记录、算了。

## 项目名称归一化
- `慈溪 VOCs`、`慈溪vocs`、`家电 VOCs`、`家电产业 VOCs` -> `慈溪市家电产业 VOCs 智治捷溯项目（二期）`
- `镇海`、`镇海二期`、`数字治气`、`镇海数字治气` -> `镇海数字治气二期项目`
- `余姚`、`余姚二期` -> `余姚二期`
- 无法识别 -> `待确认`

项目名称为 `待确认` 时，必须列入【需要确认 / 补充的字段】。

## 事项类型
只能选择：需求确认、Bug修复、数据接入、等保整改、账号权限、原型设计、文档材料、客户沟通、上线部署、验收推进、其他。

语义优先，不只看关键词：
- 接口、字段、库表、数据同步、数据清洗、数据编目、ODS、ETL、数据底座 -> 数据接入
- 等保测评、漏洞、整改、安全问题、测评报告、渗透、弱口令、漏洞扫描 -> 等保整改
- 页面、原型、UI、低保真、高保真、交互、产品流程、设计图 -> 原型设计
- 账号、权限、角色、菜单、登录、用户配置 -> 账号权限
- 系统异常、报错、bug、功能不可用、数据查不到、页面显示异常 -> Bug修复
- 上线、部署、发版、IRS、代码工厂、服务器环境 -> 上线部署
- 验收、签字、确认单、试运行、验收材料、验收报告 -> 验收推进
- 方案、制度、汇报、周报、说明书、PPT、材料编写 -> 文档材料
- 需求边界、需求口径、功能范围、是否纳入本期、客户确认 -> 需求确认
- 只有沟通过程、无明确交付物 -> 客户沟通
- 无法判断 -> 其他

出现“业主说、电话沟通、微信沟通、会议沟通”时，仍要看沟通内容的核心交付物。

## 当前状态
只能选择：待处理、进行中、待确认、已完成、阻塞、暂缓。

- 已完成、已提交、已修复、已确认 -> 已完成
- 有人正在处理，且无明显卡点 -> 进行中
- 需要某方确认后才能继续 -> 待确认
- 尚未开始，需要用户后续推进 -> 待处理
- 因资源、接口、权限、人员、外部反馈等原因无法继续 -> 阻塞
- 明确先不做、后面再说、下阶段处理 -> 暂缓
- 不确定时默认待处理，并提示为系统推断

如果状态为阻塞，提醒用户确认是否需要上报。

## 优先级
只能选择：高、中、低。

- 今天、明天、周五前、本周必须、马上、尽快、影响验收、影响上线、领导要、业主要、卡节点、月底前、必须完成 -> 高
- 有明确截止时间但不是特别紧急 -> 中
- 无明确截止时间，只是普通记录 -> 低
- 默认 -> 中

## 责任方与协同方
责任方是下一步主要推动方，不是文本中出现的人。

- 我整理、我确认、我催办、我协调、我推进 -> 我方
- 开发修复、开发实现、开发配置、开发排查 -> 开发
- 业主确认、业主反馈、业主提供资料 -> 业主
- 第三方提供接口、字段、数据、文档 -> 第三方
- 运维部署、配置账号、服务器、环境 -> 运维
- 等保公司出报告、测评、复测 -> 等保公司
- 不确定 -> 待确认

协同方可包含：开发、业主、第三方、等保公司、运维、产品、测试、领导。多个用顿号分隔。

## 截止时间
日期格式固定为 `YYYY-MM-DD`。当前日期以系统日期为准，按 `Asia/Shanghai` 或 `Asia/Taipei`。

需支持：今天、明天、后天、本周一到本周日、下周一到下周日、本周五、周五前、月底、5月1日前、4月30日前、五月一号前、四月底前。

相对日期推断出的截止时间置信度为中，并必须在确认区提醒用户确认。无截止时间则留空。表达模糊如“最近、这两天、尽快”且明显需要期限时填写 `待确认`。

## 留痕
留痕方式默认 `微信群`。

- 电话沟通、微信沟通、群里说、项目群 -> 微信群
- 邮件、发邮件、邮件确认 -> 邮件
- 会议、开会、纪要、现场会 -> 会议纪要
- 禅道、Jira、任务单 -> 禅道/Jira
- 代码工厂、IRS -> 代码工厂
- 文件、材料、附件、文档 -> 文件记录

留痕位置默认 `待补充`。不得编造微信群、时间、邮件标题或纪要编号。为 `待补充` 时必须提醒用户补充。

## 备注
备注必须保留原始输入，格式：`原始记录：……`

## 表头校验
写入前必须检查表头与目标字段顺序完全一致。不一致时不得写入，并输出当前表头与期望表头。

## 编号生成
确认写入时读取已有编号，跳过异常编号，最大合法编号 + 1。格式为 `SX-0001`。草稿阶段编号始终为 `待生成`。
